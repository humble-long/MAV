**第 5 段：本文工作概述**

# 3 尺度律工具增强的图路径推理（约 3 页 — 创新点 2）

## 3.1 总体框架

### 3.1.1 设计动机
- 工程设计的查询需要定量校验（"30 km 续航需要多大重量？"）
- 纯 LLM 无法保证物理一致性
- 纯 KG 检索无法生成自然语言推理过程
- **解决方案**：LLM 在 KG 上做路径推理 + 沿路径调用物理公式工具

### 3.1.2 推理范式
借鉴 ToG [11] 的 LLM-on-Graph beam search 范式与 RoG [12] 的关系路径规划，扩展为支持**工具调用**的混合推理：

```
查询 → 锚定起始实体 → LLM 规划候选关系路径
     → 沿路径检索 KG 子图 → 必要时调用物理工具计算
     → LLM 综合 KG 事实 + 工具结果 → 生成答案 + 推理链
```

## 3.2 物理工具库设计

### 3.2.1 4 个核心工具
| 工具名 | 输入 | 输出 | 公式来源 |
|---|---|---|---|
| `hassanalian_weight` | 任务约束（航程、载重、扑频）| 起飞重量估算 | Hassanalian 2017 [18] |
| `shyy_scaling_law` | 重量 | 翼展、翼面积、展弦比、扑频 | Shyy 2013 [19] |
| `strouhal_check` | 扑频、扑幅、飞行速度 | Strouhal 数 + 是否在合理区间 [0.2, 0.4] | 推导自 [19] |
| `reynolds_check` | 翼弦、飞行速度 | 雷诺数 + 流态判别 | 标准定义 |

### 3.2.2 工具调用接口
- 基于 Function Calling 协议（OpenAI Tool Use 标准）
- 每个工具有标准化的 JSON Schema 描述
- LLM 在推理时自主决定何时调用

## 3.3 路径推理算法

### 3.3.1 算法伪代码

```
Algorithm 1: BioBridge-Path-Reasoning
Input: query Q, KG G, LLM M, tool set T, beam width k, max depth D
Output: answer A, reasoning path P

1: entities = LinkEntities(Q)              # 锚定 KG 起点
2: paths = [{entities, depth=0}]
3: while depth < D and not Terminate:
4:     candidates = []
5:     for each path in paths:
6:         relations = M.PlanRelations(path, Q)  # LLM 规划候选关系
7:         for each rel in relations:
8:             new_nodes = G.Traverse(path.tail, rel)
9:             score = M.Score(new_nodes, Q)
10:            if M.NeedsTool(path, Q):
11:                tool_result = T.Call(path)      # 调用物理工具
12:                path.toolResults.append(tool_result)
13:            candidates.append(path + new_nodes)
14:    paths = TopK(candidates, k)
15:    if M.IsAnswerable(paths):
16:        break
17: A = M.Generate(query=Q, paths=paths, tools=path.toolResults)
18: return A, paths
```

### 3.3.2 关键设计点
- **关系规划**：每步由 LLM 在当前节点的可达关系中选 Top-k（避免遍历爆炸）
- **工具触发**：LLM 自主判别（如检测到任务约束类查询自动触发 hassanalian_weight）
- **多跳深度**：默认 D=4，覆盖 80% 以上 FWMAV 设计查询

## 3.4 应用模式

### 3.4.1 知识查询模式
**例**："鸽子和隼的扑频差异在哪？"
- 推理路径：Pigeon → frequency_range → [3-5 Hz] | Falcon → frequency_range → [4-6 Hz]
- 工具调用：无
- 输出：对比表格 + 文字解释 + 引用文献

### 3.4.2 推荐精排模式
**例**：对张量分解返回的 Top-10 候选，逐个调用 hassanalian_weight 验证物理可行性，淘汰不合理样机，输出 Top-3
- 推理路径：每个候选样机的关键性能链
- 工具调用：hassanalian_weight、shyy_scaling_law、strouhal_check
- 输出：Top-3 + 物理校验报告

# 4 基于张量分解的方案候选检索（约 2 页 — 创新点 3）

## 4.1 设计动机
- 直接对 50+ 历史样机做物理校验代价高
- 引入"粗筛 + 精排"两阶段：先用张量嵌入快速召回 Top-10
- 张量分解的 latent factor 可解释为"性能-参数耦合模式"

> **借鉴**：Jia 2021 [8] 在冲压模具设计领域的 3 阶张量；本文扩展至 4 阶张量。

## 4.2 4 阶张量构建

### 4.2.1 维度定义
张量 𝒳 ∈ ℝ^{N₁ × N₂ × N₃ × N₄}：
- 维度 1：FWMAV 样机（N₁ ≈ 50）
- 维度 2：组件类型（N₂ ≈ 20）
- 维度 3：性能指标（N₃ ≈ 10）
- 维度 4：任务场景（N₄ ≈ 5）

### 4.2.2 元素填充
- 𝒳[i, j, k, l] = 样机 i 在任务 l 下、装载组件 j 时、性能指标 k 的归一化值
- 缺失值用 KG 中的"父类节点平均值"填充（避免稀疏）

### 4.2.3 归一化
- 数值属性按 z-score 归一化
- 类别属性独热编码后再投影

## 4.3 CP 分解

### 4.3.1 分解形式

`𝒳 ≈ Σ_{r=1}^R u_r ⊗ v_r ⊗ w_r ⊗ s_r`

其中：
- u_r ∈ ℝ^{N₁}：样机 latent factor
- v_r ∈ ℝ^{N₂}：组件 latent factor
- w_r ∈ ℝ^{N₃}：性能 latent factor
- s_r ∈ ℝ^{N₄}：任务 latent factor
- R = 8（潜在维度，由交叉验证确定）

### 4.3.2 实现
- TensorLy 库 + ALS（交替最小二乘）求解
- 求解时间约 O(N₁N₂N₃N₄ R) ≈ 5 万步运算，秒级

## 4.4 任务约束 → 嵌入查询

### 4.4.1 查询向量构造
用户输入任务约束 → 抽取 N₃（性能维度）+ N₄（任务维度）→ 拼接为查询向量 q ∈ ℝ^{2R}

### 4.4.2 KNN 检索
- 计算 q 与每个样机嵌入 u_i ⊕ task_proj 的余弦相似度
- 取 Top-10 作为候选

### 4.4.3 latent factor 可解释性
- 用 silhouette analysis 解释每个 latent factor 的语义（如 r=2 对应"高扑频 + 轻翼载 + 短续航"模式）
- 输出推荐时附带"该样机匹配的设计模式"

# 6 结论（≤ 0.5 页）

> **航空学报要求**：分点总结，只写结论，不重复背景方法。

1) 提出了仿生飞行器领域的双层知识图谱构建方法，通过引入生物原型层和 4 类仿生映射关系，实现了生物-工程跨域知识的统一表达，构建的知识图谱包含约 N1 个工程样机节点与 N2 个生物原型节点。

2) 提出了尺度律工具增强的图路径推理框架，将 Hassanalian 翼型公式与 Shyy 尺度律封装为大语言模型可调用的物理工具，使方法在多跳问答任务上的 Faithfulness 较纯大语言模型基线提升了 X 个百分点。

3) 引入 4 阶张量分解的方案候选检索，与路径推理形成"粗筛-精排"协同框架，方案推荐 NDCG@5 较纯向量检索基线提升了 Y 个百分点，物理可行性通过率达到 Z%。

4) 在自建 FWMAV-QA 数据集上的消融实验验证了三个创新点对最终性能的独立贡献，所提方法可为仿生飞行器概念设计阶段的可解释决策支持提供有效手段。后续工作将考虑引入多模态图表数据抽取以进一步丰富知识图谱。
